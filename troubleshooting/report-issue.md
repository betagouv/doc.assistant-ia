# Signaler un problème

Rencontrez-vous un bug, une erreur ou un comportement inattendu ? Signalez-le nous pour que nous puissions investiguer et corriger le problème rapidement.

## Avant de signaler

Vérifiez d'abord si votre problème est déjà documenté :

1. **[Consultez le guide de dépannage](troubleshooting.md)** - Solutions aux problèmes courants
2. **[Parcourez la FAQ](faq.md)** - Réponses aux questions fréquentes
3. **Essayez les solutions proposées** avant de signaler

## Comment signaler un problème

### Méthode 1 : Via le formulaire intégré (recommandé)

Dans l'application Assistant IA :

1. Cliquez sur le bouton **"Faire un retour"** (généralement en bas à droite ou dans le menu)
2. Sélectionnez **"Signaler un problème technique"**
3. Remplissez le formulaire avec les informations demandées
4. Envoyez votre signalement

### Méthode 2 : Via le formulaire en ligne

Accédez au formulaire dédié : [https://formulaire.beta.numerique.gouv.fr/r/assistant](https://formulaire.beta.numerique.gouv.fr/r/assistant)

### Méthode 3 : Via Tchap

Rejoignez le canal dédié et décrivez votre problème :
[https://tchap.gouv.fr/#/room/!eAHyPLdVHMxNhKAbaC:agent.dinum.tchap.gouv.fr](https://tchap.gouv.fr/#/room/!eAHyPLdVHMxNhKAbaC:agent.dinum.tchap.gouv.fr)

### Méthode 4 : Par email

Envoyez un email à : **support-assistant@numerique.gouv.fr**

## Informations à fournir

Pour que nous puissions traiter votre signalement efficacement, **merci de fournir un maximum d'informations** :

### 📋 Informations obligatoires

| Information | Exemple | Pourquoi c'est important |
|-------------|---------|--------------------------|
| **Description du problème** | "L'Assistant ne répond pas quand je clique sur Envoyer" | Nous permet de comprendre le problème |
| **Étapes pour reproduire** | 1. Ouvrir une nouvelle conversation 2. Écrire "Bonjour" 3. Cliquer sur Envoyer | Nous permet de reproduire et corriger |
| **Comportement attendu** | "L'Assistant devrait répondre avec un message de salutation" | Savoir ce qui devrait se passer |
| **Comportement réel** | "Rien ne se passe, pas de réponse" | Comprendre l'anomalie |

### 📊 Informations techniques utiles

| Information | Comment l'obtenir | Exemple |
|-------------|-------------------|---------|
| **Navigateur** | Menu Aide > À propos | Chrome 125.0.6422.60 |
| **Système d'exploitation** | Paramètres système | Windows 11, macOS Sonoma 14.4 |
| **Appareil** | - | ordinateur portable, iPhone 15 |
| **Date et heure** | - | 29/05/2026, 14:30 |
| **URL de la page** | Barre d'adresse | https://assistant.numerique.gouv.fr/chat/abc123 |

### 📎 Pièces jointes utiles

Vous pouvez joindre :
- **Capture d'écran** du problème (avec l'URL visible si possible)
- **Vidéo** montrant les étapes pour reproduire (max 2 min)
- **Fichier** qui pose problème (si applicable)
- **Logs** (pour les utilisateurs techniques)

{% hint style="info" %}
**Pour les captures d'écran :** N'incluez **pas** de données sensibles ou personnelles. Floutez les informations confidentielles avant l'envoi.
{% endhint %}

## Catégories de problèmes

Sélectionnez la catégorie qui correspond le mieux à votre problème :

### 🔌 Problèmes de connexion
- Impossible de se connecter via ProConnect
- Erreur d'authentification
- Session qui expire trop rapidement
- Problème de déconnexion intempestive

**Informations supplémentaires à fournir :**
- Message d'erreur exact
- Heure du problème
- Avez-vous essayé avec un autre navigateur ?

### 💬 Problèmes de conversation
- L'Assistant ne répond pas
- Réponse incomplète ou coupée
- Erreur lors de l'envoi d'un message
- Problème d'historique des conversations

**Informations supplémentaires à fournir :**
- Texte du message qui pose problème (si applicable)
- Longueur de la conversation
- Modèle utilisé

### 📄 Problèmes de documents
- Impossible de téléverser un fichier
- Erreur lors de l'analyse d'un document
- Document non reconnu ou illisible
- Problème avec un type de fichier spécifique

**Informations supplémentaires à fournir :**
- Type de fichier (PDF, DOCX, etc.)
- Taille du fichier
- Contenu du document (description générale)

### 🌐 Problèmes de recherche web
- Bouton de recherche grisé
- Pas de résultats de recherche
- Résultats incorrects ou non pertinents
- Erreur lors de la recherche

**Informations supplémentaires à fournir :**
- Requête de recherche
- Modèle utilisé
- Réglages de recherche (manuelle ou intelligente)

### 🎨 Problèmes d'interface
- Affichage incorrect
- Bug visuel
- Problème de navigation
- Éléments manquants ou mal placés

**Informations supplémentaires à fournir :**
- Capture d'écran du problème
- Résolution de votre écran
- Navigateur utilisé

### 🐛 Autre bug technique
- Comportement inattendu
- Erreur système
- Plantage de l'application
- Problème non listé ci-dessus

**Informations supplémentaires à fournir :**
- Description détaillée du comportement
- Étapes exactes pour reproduire
- Message d'erreur complet (le cas échéant)

## Niveaux de priorité

Votre signalement sera traité selon son niveau de priorité :

| Priorité | Temps de réponse cible | Exemples |
|----------|------------------------|----------|
| **🔴 Urgent** | < 2 heures | Fuite de données, accès bloqué pour tous les utilisateurs |
| **🟡 Élevé** | < 24 heures | Problème majeur empêchant l'utilisation, données corrompues |
| **🟢 Normal** | < 72 heures | Bug mineure, problème de fonctionnalité secondaire |
| **🔵 Faible** | < 1 semaine | Suggestion d'amélioration, problème cosmétique |

{% hint style="warning" %}
**Problèmes urgents :** En cas de suspicion de **fuite de données** ou de **violation de sécurité**, contactez **immédiatement** votre référent IA ou l'équipe DINUM par téléphone ou via les canaux sécurisés.
{% endhint %}

## Que se passe-t-il après le signalement ?

1. **Accusé de réception** : Vous recevrez une confirmation par email dans les 24 heures
2. **Analyse** : Notre équipe technique investigate le problème
3. **Contact** : Nous pourrions vous recontacter pour des informations complémentaires
4. **Résolution** : Nous travaillons à corriger le problème
5. **Fermeture** : Vous serez informé lorsque le problème est résolu

**Temps de traitement moyen :** 3 à 5 jours ouvrés (variable selon la complexité)

---

## Conseils pour un signalement efficace

✅ **À faire :**
- Soyez **précis** et **détaillé**
- Fournissez des **étapes claires** pour reproduire
- Joignez des **captures d'écran** ou **vidéos**
- Mentionnez votre **navigateur** et **système d'exploitation**

❌ **À éviter :**
- "L'Assistant ne marche pas" (trop vague)
- "J'ai un problème" (sans détails)
- Signaler des **problèmes déjà connus** (vérifiez le statut du service)
